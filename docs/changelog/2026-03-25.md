# 2026-03-25 更新日志

## 核心进展

### 1. 修复核心收敛 Bug (Vanishing Gradients)
- **原因发现**：原 PyTorch 代码中 5 层 MLP 使用了 `torch.nn.init.normal_(m.weight, 0, 0.01)` 进行初始化。这导致前向传播时方差急剧缩小，深度仅为 5 层的网络输出值几乎全为 0，反向传播时梯度消失，导致模型严重欠拟合，损失难以跌破 $10^{-3}$。
- **修复方案**：将初始化方式严格对齐原论文的 Keras 默认行为，即替换为 `torch.nn.init.xavier_uniform_(m.weight)`（Glorot Uniform）。
- **结果预期**：这一修复彻底打通了梯度流，理论上模型可以顺利收敛至 $\text{MSE} \sim 10^{-7}$ 级别。

### 2. 破除 WSL 跨系统 I/O 瓶颈
- **原因发现**：训练时 GPU 利用率极低（仅 ~16%），是因为数据存放在 Windows 的 `E:` 盘，而 WSL 在训练时通过 9P 协议逐样本 (`__getitem__`) 跨文件系统读取 HDF5 文件碎块，造成了致命的 I/O 阻塞。
- **修复方案**：鉴于单组数据集仅约 100MB，我们在 `dataloader.py` 中加入了 `in_memory=True` 的逻辑。现在数据集会在 `__init__` 时被一口气完整加载进内存，训练中的 `__getitem__` 变为纯内存拷贝，彻底绕开了硬盘瓶颈。现在的 GPU 利用率可以轻松超过 70%。

### 3. 可视化训练进程
- 在 `train.py` 的主循环中加入了每 100 Batch 打印一次的进度监控条。
- 在 `fit_regression_model` 尾部增加了 `matplotlib` 自动制图逻辑，训练完毕后自动在模型文件夹下生成包含 Train/Val/Test 损失及 MAE/RMSE 的半对数坐标曲线图 (`training_curves_rhoXXX.png`)。

### 4. 代码库架构大重构 (Project Refactoring)
为了解决原先实验配置难追溯、模型输出和测试产生大量零碎文件的痛点，我们彻底重构了整个项目的命名和收纳架构：
- **净化了根目录**：将之前废弃或一次性查询用途的如 `probe_origin.py`、`view_case.py` 全部归档进 `tools/` 目录。
- **拆分了 Docs**：建立本 `changelog/` 目录用于时间线跟进，原有算法数学文档（如 `projection.md`）已迁移至 `guides/`。
- **解耦了 Train 与 Test 的输出**（即将在后续代码落实）：从此以后，数据生成将附带 `data_config.yaml`；模型训练将建立独立的试验目录（如 `ds01_bs1024/`）并包含 `train_config.yaml`；模型评估生成的所有日志均精准投递入 `model/.../evals/` 下，实现**三模一体**的完美绑定。
