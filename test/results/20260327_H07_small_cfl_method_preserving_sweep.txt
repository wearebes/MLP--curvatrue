Experiment: H07 small-CFL / method-preserving sweep

Date: 2026-03-27
Goal:
- keep cfl in the 0.5 to 0.6 range
- avoid large parameter drift
- test whether a more standard sign-function method can recover most of the paper alignment

Protocol:
- phi0 mode: formula_phi0
- steps: 5, 10, 20
- cases: smooth_256, smooth_266, smooth_276, acute_276
- numerical ratio = mean( MSE_current / MSE_paper_numerical )
- origin ratio = mean( MSE_current / MSE_paper_model )

Method family B: phi0-gradient-aware sign smoothing

Definition under test:
- S(phi0) = phi0 / sqrt(phi0^2 + (eps_sign_factor * h * |grad(phi0)|)^2)
- this is closer to the standard non-SDF-aware smoothing than the current plain-h version

Interpretation:
- gradient-aware smoothing is a real method improvement over the current plain-h sign smoothing
- but with eps_sign_factor still fixed at 1.0 and cfl still capped at 0.6, it does not get close enough to the paper tables

Small-range sweep inside the gradient-aware family

cfl=0.60 eps=1.40 sign=dynamic_phi:
- numerical ratio = 1.256
- origin ratio = 2.655

Interpretation:
- if we insist on keeping cfl <= 0.6 and eps_sign_factor <= 1.5, the best method-preserving variant is:
  phi0-gradient-aware sign + dynamic_phi + cfl=0.6 + eps_sign_factor=1.5
- this already brings numerical close to paper, but origin is still about 2.23x the paper model MSE on average

Practical takeaway:
- pure method repair with very small parameter motion helps a lot, but does not fully close the paper gap
- the smallest practical compromise that worked much better in earlier sweeps was still:
  dynamic_phi + cfl=0.6 + eps_sign_factor about 2.0 to 2.25
- so there is a real tradeoff:
  1. minimal parameter change -> better method story, but still visible paper gap
  2. moderate parameter change -> much closer paper alignment
